My Learning

Abstract

湖南工业大学研究生在读，研究方向：深度学习、图像识别，语言为Python/Java。

在我的认知里，一个优秀的人的必要品质是持续不断的学习，这在于工科方面尤为重要。当学习间断的时候，就是被社会摈弃的时候。从更深层次上说，这是探索人活着的意义的能力之一。

所以在我看来，学习的驱动是有层次的。一般来说，人生观>责任>兴趣>目的，这三点需要建立好才能完成真实有效的学习，但是顺序并不是完全固定的。但可以确切说，靠简单直接的目的去学习，是学不好的。

很遗憾，我在22岁才意识到学习的意义。
很幸运，我在22岁就发现了学习的魅力。

Java

关于学习Java，我是报以公众号后端开发接口为目的的。因为我本身是一位拥有三万粉丝的公众号号主，所以便想通过学习Java后端进行开发。 而此时我已经研二了，这意味着我要么选择实习，要么选择学Java，甚至要放弃很多的工作机会。

为了不后悔，我选择了学习Java，继而学习多线程、集合等，包括类加载，垃圾回收机制，学习使用MySQL，以及与Java交互接口的JBDC，MySQL事务与隔离、批处理、检索等，自学HTML，CSS并制作了这个静态网页 接下来又了解UCP/TCP模型结构与特点，熟练使用Socket操作指令。

而事实上，这些远远不够，我还要学习spring框架等。 好像有点来不及了，工作迫在眉睫。其实我不太后悔，反正人生在世，总不能总是浑浑噩噩。

况且，我才24岁。

Python

人工智能是神秘的，无论是抽象感知上还是具体实施上。

我是自动化学科出生，可以说计算机只是匮乏的可怜。而学习人工智能方向的Python，可以直接说是面向API学习了，这就导致我故步自封了。 而在学习过程中，只是在自己的一亩三分地上耕耘，胜利的果实自然而然的稀缺。

框架，是我听到编程上做多的，也是最恐怖的词。其实代码意味着无限的可能性，实现的方式也是千变万化，而在工程实现上，我们往往是由一套规范，一套框架，这是无法避免的。 我总觉得我骨子里像个文科生，在工科上没什么能力，但满脑子总是一些浪漫主义自由主义的事情。

但是，Python对我来说，是启蒙，是让我发现代码之美的老师。

只是突然感叹，为了生活，为了工作，我还能坚持多久。

我的学习方式

我最喜欢读课外书，尤其是鲁迅、余华、王小波，甚至卡夫卡、叔本华等。

但是在知识向的学习中，我更倾向于看视频和记笔记。多维度的学习自然是最好的，因为灌输的信息是最多的，但同时也会有很直接的弊端。比如看视频相对于看书，趣味性和信息量都会大大增加，但是弊端也尤为明显，就是信息量过多，导致的信息流失。

所以停一停，记一下笔记，建立属于自己的具象的数据库，一定是事半功倍。

我的学习策略

关于如何学习。我会首先制定一个由学习目的驱动的属于自己的学习策略，自下而上完善，在学习中不断调整。而其中最重要的是正视自己的缺点，并作出切实可行的符合自己特色的规划。

对于我本人，一个不算多聪明的人来说，始终认为人性的缺点很多时候不需要刻意挑战，这一点很重要。如果你喜欢独处，那就去独处；如果你兴趣持续时间短，那就在最短的时间内压榨出自己所有的潜力。

无论，爱上学习本身，发现学习本身的魅力才是我们的一生的必修课。

我认为的学习品质

这个品质我和别人的想法都不太一样，我认为这个重要的学习品质是——审美。

审美的作用域不仅仅在生活上，其实也在学习上。代码的审美，艺术的审美，还是自然的审美，当美成为了一种追求，那么人本身就会成为美的一部分。

其实在科学领域上，欧拉公式就被誉为世界上最美的公式，而所有的科学终点都是哲学，那么哲学和审美自然就有很大的重合。

所以，我觉得很有必要提高自己审美的能力，对无论是美丽的，丑陋的，提高对美的兼容性，完善对美的掌控能力。

总而言之

其实作为一个真正的学习小白，说这些本来没有什么意义。

但我总是想着，人总要进步的。每一个阶段都有自己的人生观和理由，记录下来，也是个很酷的一件事情。

不管怎样，做现在的自己，就是最好的。
